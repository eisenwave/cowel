# Bindings

This directory contains non-core integration layers for COWEL.

- `native/` contains the native CLI wrapper sources.
- `node/` contains the Node.js wrapper,
  the WASM runner,
  and Node-based test suites.
- `lsp/` contains the LSP server implementation,
  built natively and as WASM.
- `test/` contains integration test fixtures for bindings.

## Bindings Test Fixtures

`bindings/test/` stores CLI integration fixtures.
Each `*.cow` file has a matching `*.cow.txt` file,
which contains the expected CLI output.

`bindings/test/lsp/` stores LSP integration fixtures.
Each immediate subdirectory is one test suite.
A suite must include:

- `.cowel_config.json`
- at least one `*.cow` source file
- `input.json`
- `output.json`

`input.json` and `output.json` are JSON arrays of unframed JSON-RPC payload objects.
They intentionally do not contain `Content-Length` headers.
The Node test harness performs framing/parsing during execution.

## WASM Build Contract For Node Tests

The Node integration tests are expected to run against artifacts generated by the CMake WASM build.
The intended path is to configure CMake with an Emscripten toolchain,
build target `cowel-npm`,
build target `cowel-lsp-wasm`,
and then run `npm test` from `bindings/node`.

`cowel-npm` must generate:

- `build/npm/cowel.wasm`
- `build/npm/cowel.js`
- transpiled Node test files under `build/test/`

`cowel-lsp-wasm` must generate:

- `build/lsp-wasm/cowel-lsp.wasm`
- `build/lsp-wasm/lsp-wasm-runner.js`

If those artifacts are missing,
Node integration tests are expected to fail,
because that indicates the WASM build contract was not satisfied.
